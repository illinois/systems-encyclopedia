# Binary

The binary number system is the most fundamental way of representing information in all computer systems. Understanding binary is fundamental to understanding how computers process information and how data is structured in every computer program. Here, we'll provide a brief overview of how binary works, and cover some basic arithmetic operations over binary numbers.

## Overview

Binary is a system of representing number with a **base** (or **radix**) of 2. This means for every number in binary, each digit consists of one of two possible symbols (either a 0 or 1). In computing, binary is the only true form of data representation -- all other forms of representing data are simply abstractions. Computers use binary because it is the system that is best able to mimic the physical behavior of the components that make up a computer.

In computing, any sequence of binary digits is commonly referred to as a **bit string** or **binary string**. Each individual digit of the binary string is referred to as a **bit**. For any binary string, the leftmost digit is called the **most significant bit**, and the rightmost bit is referred to as the **least significant bit**. 

Binary numbers can be also either **signed** or **unsigned**, where unsigned numbers are guaranteed to be non-negative, but signed numbers can be negative. For signed numbers, the most significant bit is called the **sign bit** &mdash; if the sign bit is set to 1, the number is negative. In many programming languages, numerical data types are fixed-size, meaning that the underlying binary representation of that data type cannot be more than a certain number of bits long. In C, for example, every `int` is guaranteed to be exactly 32 bits long. In practice, because `int` is a signed data type, the max and min values of an `int` are 2<sup>31</sup> - 1 and -2<sup>31</sup> respectively (since 31 bits are for indicating number magnitude, and the last bit is the sign bit). If an arithmetic operation would exceed the maximum or minimum value of the data type, then a condition called **overflow** will occur (explained below).

## Binary arithmetic

### Addition

Adding binary strings is similar to the way you may have added decimal numbers in grade school. Specifically, addition involves the use of a **carry bit**, just like you would use a carry digit when doing digit-wise addition. To add two binary strings together, we use the following technique:
- Write both binary strings together columnwise.
- Starting with the least significant bit, add the binary strings bitwise as follows:
    - If both operand bits are 0 and the carry bit is 0, the resulting bit in the sum will be 0
    - If both bits are 1 and the carry bit is 0, the resulting bit will be 0, and the carry bit will be set to 1
    - If both bits are 1 and the carry bit is 1, the resulting bit will be 1, and the carry bit will remain 1
    - If one operand bit is 0, another is 1, and the carry bit is 0, the resulting bit is 1
    - If one operand bit is 0, another is 1, and the carry bit is 1, the resulting bit is 0 and the carry bit remains 1

In cases where the first operand is also negative (e.g. -12 - 13), both operands should be converted into their two's complement form and then added. In the case where the second operand is negative (e.g. 12 - (-13)), regular addition should be used.

It is important to note that the two's complement of a number is not strictly just the negative form of a number (i.e. the two's complement of 33 is not -33). So, numbers in two's complement form need to first be converted out of two's complement form to be converted to decimal. Because two's complement is reversible, simply apply the process used to convert a number to its two's complement form again to get its regular form.

### Overflow

For fixed-size binary strings, there are cases where an arithmetic binary operation can lead to **overflow**, which occurs when a binary string is unable to hold a certain decimal value with the number of bits it is restricted to. To understand overflow, let's look a simple case of adding two unsigned 4-bit binary numbers:

<img src="../static/binary/binaryaddition.gif"></img>

In the case where our carry bit is set to 1 but we have exceeded the length of the two binary strings, the last digit becomes discarded completely! In practice, overflow leads to getting results that are completely different from what we expected. So, whenever adding fixed-size data types, it is important to make sure to watch for overflow.

For any two fixed-size binary strings, overflow is achieved under the following conditions:
- For unsigned binary strings, overflow has occurred if the carry bit still one after the entire addition of the string
- For signed binary strings:
    - If the operands have different sign bits, overflow can never occur (think about why this is the case!).
    - If the operands have the same sign bit, overflow has occured if the sign bit of the result is different from the operands.
